Step 01 · Trigger
Source File Receipt
Billing source data is received by the SPOC team. Files arrive through multiple channels depending on the billing type.
Email from ELM Team
Email from Business Team
Client Portal (self-service)
Note: Source files may arrive from ELM, Business Team via email, or directly via the client portal for certain billing types. Channel may vary per transaction.
Step 02 · SPOC Action
Email & Attachment Review by SPOC
The designated SPOC (Single Point of Contact) reviews the incoming email along with all attached documents to identify the billing context and determine the applicable billing type from the 13 available categories.
SPOC assesses the email body, attached files, and any client references to classify the transaction correctly before proceeding.
Step 03 · Classification
Billing Type Determination — 13 Types
SPOC identifies the applicable billing type. Each type follows a distinct template and processing path.

Template Notice: The Invoice Draft Template structure differs based on the transaction/billing type. SPOC must select and populate the correct template variant specific to the identified billing type before proceeding to preparation.
Step 04 · Preparation
Invoice Draft Sheet Preparation
The SPOC prepares the Invoice Draft Sheet using the billing-type-specific template. Line items, rates, quantities, tax fields, and entity details are populated from the source data.
The same standard template format and structure is used across all billing types — however, the values within the fields are calculated and derived based on the specific transaction type being processed.
Step 05 · Quality Control
First-Level QC Check (Validation)
The drafted invoice sheet undergoes a first-level QC review. The reviewer validates data accuracy, correct billing type template usage, calculations, tax compliance, and completeness before escalation.
Checks include: arithmetic accuracy · tax codes · entity details · billing-type-specific fields · source data match · policy compliance.
Step 06 · Approval
Manager Approval
The QC-validated draft sheet is shared with the Manager for final review and approval. The Manager authenticates the invoice details, authorises processing, and confirms readiness for SAP submission.
Manager sign-off is mandatory before any SAP data push. Rejected drafts are returned to the SPOC for correction.
Step 07 · SAP Entry
File Push to SAP
Upon manager approval, the invoice data is entered / pushed into the SAP system by the SPOC. The transaction is posted against the relevant cost centre, entity, and billing configuration.
SAP module used depends on billing type. Data is entered manually or via batch upload based on volume.
Step 08 · System Output
SAP System Response — Process Complete
SAP processes the posted transaction and returns the confirmation response. The workflow is considered complete upon successful receipt of both identifiers.
